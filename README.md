# MIYP Bank (Money In Your Pocket)

> **Small Bank, Big Trust.**

Welcome to my first OOP-based banking system built in Python!

This isn't just another mini project. It's a project that taught me how Object-Oriented Programming actually works in a real application.

When I started building MIYP Bank, I thought the difficult part would be writing the code. Surprisingly, the real challenge was understanding **the flow of the program**.

Questions like...

- Why is the object created after account creation?
- Why does login happen outside the class?
- Where should the dashboard loop be?
- How do getters and setters actually work?
- Why do we use properties instead of directly accessing attributes?

...were much harder than writing the syntax itself.

Instead of copying a tutorial, I spent several days designing, debugging, asking questions, fixing logic, and improving the project step by step.

Today, this project represents not only a banking system but also my growing understanding of Python and OOP.

##  What This Project Taught Me

More than writing code, this project taught me how to think while writing code.

Some of my biggest learnings were:

- Designing the flow before writing the logic.
- Understanding why methods belong inside a class.
- Learning how getters and setters protect data.
- Knowing when to use properties instead of directly accessing attributes.
- Breaking a large problem into smaller functions.
- Debugging my own mistakes instead of immediately searching for solutions.

Every improvement made this project a little better and made me a little more confident.
